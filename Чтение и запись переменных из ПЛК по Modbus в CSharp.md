## Чтение и запись переменных из ПЛК по Modbus в C#-приложении

### Ключевые идеи
- Modbus TCP — самый простой протокол для связи ПЛК ↔ C# (заработал с первого раза, без допустановок в CODESYS)
- Modbus оперирует только 16-битными регистрами, макс 125 за один запрос, всего 4096 в CODESYS
- Значения > 16 бит (32/64) распределяются по нескольким регистрам → нужен буферный массив
- Заголовок в регистрах: тип проекта + версия — чтобы не читать «чужую» карту переменных
- Класс `ModbusVariable`: имя, тип C#, адрес, размер в регистрах, значение (INotifyPropertyChanged)
- Generic `ModbusVariable<T>` красиво, но на практике тип известен только в рантайме → проще `object`
- Библиотека: NModbus (NuGet), альтернатива PL.Modbus (мультизапросы)

### Резюме
> Практическое руководство по построению типизированной обёртки над Modbus TCP для C#. Вместо работы с «сырыми» регистрами — архитектура с заголовком проекта, картой переменных и автоматическим преобразованием типов. Подход применим к любому ПЛК с поддержкой Modbus.

### Связь с моей работой
- **Проекты (АСУТП):** Прямое применение — интеграция SCADA/MES с ПЛК на Увелке и Зела. Можно использовать для связи 1С ↔ ПЛК через промежуточный C#-сервис. Заголовок с версией проекта решает проблему, когда в один ПЛК грузят разные программы.
- **Личное развитие:** Углубление в программирование верхнего уровня, C# для промавтоматизации

### Почему выбран Modbus (а не другие протоколы)
| Протокол | Почему не подошёл |
|----------|------------------|
| EtherNet/IP | Нет C#-библиотеки для Adapter (сервера), real-time на C# затруднён |
| MQTT | В CODESYS нет внятной документации к библиотеке |
| PLCHandler | Платное, непонятный процесс лицензирования |
| Python-скрипты CODESYS | Сложно, не все в команде знают Python |

### Особенности CODESYS + Modbus
- 4096 регистров всего
- Holding Register сбрасывается в 0 после старта ПЛК
- Массив в регистре → доступен только первый элемент
- Строки в регистрах в обратном порядке

### Архитектура
```
[ПЛК / CODESYS]
    │
    ├── Заголовок (тип проекта + версия)
    ├── Буферный массив регистров
    │     └── Переменные упакованы последовательно
    │
    ▼ Modbus TCP
    │
[C# приложение]
    ├── NModbus (чтение/запись)
    ├── ModbusVariable (типизация)
    └── Карта переменных (метаданные на стороне C#)
```
