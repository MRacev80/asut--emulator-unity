## 1. Проблема и решение

### Проблема

В АСУ ТП проектах ST-код пишется и тестируется впервые на объекте — во время пусконаладки. Это означает:

- Баги обнаруживаются под давлением времени, при живом оборудовании
- FAT (Factory Acceptance Test) проводится формально или не проводится вовсе
- Заказчик не видит работу системы до момента монтажа
- Документация пишется постфактум или не пишется совсем
- Онбординг нового инженера — только «смотри и запоминай»

### Решение

Цифровой стенд, на котором ST-код пишется, тестируется и демонстрируется до выезда на объект.

**Что даёт:**

| Кто | Что получает |
|---|---|
| Инженер-разработчик | Пишет код быстрее (Claude генерирует ST из spec), тестирует локально |
| Инженер-наладчик | Проходит FAT на стенде, приезжает на объект с проверенным кодом |
| Заказчик | Видит работу системы в Unity HMI или своей SCADA до монтажа |
| Руководитель проекта | Документация генерируется автоматически из spec + ST-кода |

### Ключевой результат

ST-код протестирован, задокументирован и показан заказчику до выезда на объект.

## 3. Архитектура

```
  Claude Code (MCP)
       │ пишет ST, компилирует, deploy
       ▼
  CODESYS Control Win V3
  ST Logic + OPC UA Server
       │
       ├──────────────────────────────────────────┐
       │ OPC UA (1 сессия)                        │ OPC UA (direct)
       ▼                                          ▼
  Python Bridge                            MasterSCADA 4
  ┌────────────────────────────┐            (заказчик)
  │ asyncua OPC UA client      │
  │ websockets server          │
  │ FAT automation             │
  │ логирование                │
  │ Claude API (опционально)   │
  └──────────┬─────────────────┘
             │ WebSocket / JSON
             ▼
        Unity HMI
        (FAT, демо)
```

### Почему такая архитектура

**OPC UA вместо Modbus:**
OPC UA даёт структурированные данные с именами тегов, push-подписки (не polling), встроенную типизацию и безопасность. Modbus — регистры без семантики, нужно вручную маппировать адреса.

**Python Bridge вместо прямого OPC UA в Unity:**
OPC UA C# SDK в Unity — проблемы с IL2CPP компиляцией, reflection, сертификатами. Python Bridge скрывает сложность OPC UA, Unity получает простой WebSocket/JSON. Дополнительный бонус: Python — единое место для FAT автоматизации, логирования, AI.

**MasterSCADA 4 напрямую к CODESYS:**
MasterSCADA 4 — нативный OPC UA клиент. Прямое подключение без посредника означает что заказчик тестирует именно то, что будет на объекте.

## 4. Компоненты

| Компонент | Роль | Запуск |
|---|---|---|
| **CODESYS Control Win V3** | ST runtime + OPC UA Server | Автозапуск как служба Windows |
| **Claude Code + MCP** | Написание ST, компиляция, deploy | `claude` в терминале |
| **Python Bridge** | OPC UA клиент, WebSocket сервер, FAT | `python python-bridge/main.py` |
| **Unity HMI** | Визуализация П&ИД, демо, FAT-стенд | Запуск Unity-билда или из редактора |
| **MasterSCADA 4** | SCADA для приёмочного теста и поставки заказчику | Отдельная установка |

## 5. Технический стек

| Слой | Технология | Обоснование |
|---|---|---|
| ST-программирование | CODESYS SP17, IEC 61131-3 | Стандарт отрасли, целевые ПЛК Увелка/Зэла |
| AI-генерация ST | Claude Code + codesys-mcp-toolkit | MCP управляет CODESYS через Scripting Engine |
| Runtime | CODESYS Control Win V3 (SoftPLC) | Разработка/тест на Windows, продакшн — реальный ПЛК |
| Протокол связи | OPC UA | Структурированные данные, push-подписки, стандарт IEC 62541 |
| Integration Hub | Python 3.11+, asyncua, websockets | Единая точка: FAT, логирование, AI, мок |
| HMI тест/демо | Unity 2022 LTS, uGUI, NativeWebSocket | Богатая 2D П&ИД визуализация |
| HMI заказчик | MasterSCADA 4 | Требование заказчика, OPC UA native |
| FAT автоматизация | Python asyncio + pytest | Скрипты тестов, авто-отчёты |
| Документация | Claude (из spec.md + ST) | Автогенерация |
| Версионирование | Git | — |

## 6. Соглашения и шаблоны

### Структура репозитория

```
/specs          — spec.md файлы по каждому объекту/блоку
/codesys        — .project файлы CODESYS
/python-bridge  — Python Bridge (asyncua + websockets + FAT)
/unity          — Unity проект
/docs           — сгенерированная документация
```

### Комментарии в ST-коде

Кириллица в CODESYS SP17 превращается в иероглифы (UTF-8 vs Windows-1251).
**Правило:** комментарии писать транслитом (латинскими буквами по-русски).

```pascal
// Vklyuchenie nasosa pri davlenii vyshe poroga
// Sbros schyotchika po signalu
// Avariynaya ostanovka
```

### Именование тегов OPC UA

```
<Объект>.<Блок>.<Переменная>
Primer: Shnek.Motor.bStart, Shnek.Motor.nSpeed
```

### Структура spec.md

```markdown
# Название блока
## Теги (таблица: имя, тип, адрес OPC UA, описание)
## Логика (условия → действия)
## Диаграмма состояний (Mermaid)
## Аварийные ситуации
```
